# Changelog

## v0.1.2

### Added

- Session-scoped attestation output persistence (in addition to `latest`)
  - `.attest_run/attestations/sessions/<SESSION_ID>/attestation.json`
  - `.attest_run/attestations/sessions/<SESSION_ID>/attestation.sig`
  - `.attest_run/attestations/sessions/<SESSION_ID>/attestation.pub`
- `audit_summary.json.workspace_files`
  - file-level write aggregation (`path`, `write_count`, `comms`, resolved `writers`, unresolved counters)
- Session correlation artifact generated by `verify --write-result`
  - `.attest_run/reports/sessions/<SESSION_ID>/session_correlation.json`
  - includes:
    - forbidden exec lineage -> files correlation
    - commit files -> writers / match-kind correlation
- Raw audit-log integrity verification (when local session state exists)
  - `attested verify` recomputes `event_root` from:
    - `audit_exec.jsonl`
    - `audit_workspace_write.jsonl`
  - compares against:
    - `event_root.json`
    - `attestation.json` integrity fields
  - fails with `AUDIT_LOG_INTEGRITY_MISMATCH` on mismatch
- WebUI cards for file-level and commit-level audit explainability
  - `Workspace Files -> Writers (Session)`
  - `Files Touched by Forbidden Exec Lineage (Session)`
  - `Commit Files -> Writers (Session)`
- E2E tamper test mode
  - `scripts/test_e2e.sh --tamper-raw-log-after-attest`
  - intentionally tampers raw audit log and verifies `AUDIT_LOG_INTEGRITY_MISMATCH`

### Improved

- WebUI now resolves and displays **selected-session** `attestation.json` (instead of always using `latest`)
- WebUI policy display (`Applied Policy`) now follows the selected session attestation snapshot
- WebUI `Attestation / Verification` card correctness
  - session-specific attestation conclusion/reason
  - summary booleans from `ATTESTED_SUMMARY` shown as supplemental verify status
  - raw-log integrity status (`PASS` / `FAIL` / `not checked`) from `ATTESTED_SUMMARY`
- WebUI readability for commit correlation
  - `Match Kind` (`exec` / `writer` / `exec+writer`)
  - hidden noisy paths in UI only (`.git`, `.attest_run`, `ATTESTED*`)

### Changed

- WebUI session correlation cards now prefer persisted JSON artifacts (`session_correlation.json`) as the source of truth
  - in-memory derivation remains as fallback for backward compatibility
- `ATTESTED_SUMMARY` now records raw audit-log integrity verification result fields
  - `audit_log_integrity_checked`
  - `audit_log_integrity_ok`
- `ATTESTATION_SCHEMA_EXAMPLES` (EN/JP) updated with:
  - `audit_summary.json.workspace_files`
  - `session_correlation.json`

### Notes

- Raw logs and audit artifacts are preserved; UI filtering hides some paths only for readability
- Session correlation remains an evidence correlation (PID/PPID lineage), not a claim that a forbidden executable directly performed the write syscall

## v0.1.1

### Added

- `attested webui` (built-in HTTPS UI) for reviewing recorded audit results
  - attestation / verification summary
  - audit summary
  - session identities (`exec` / `writer`)
  - workspace cumulative observed identities
  - session list with PASS/FAIL conclusion
- WebUI screenshots and usage guidance in top-level README / JP README
- Quickstart guidance (EN/JP) for launching and using WebUI

### Improved

- Workspace/session UX for no-argument command flows (`./attest/attested.yaml` auto-load, `.attest_run/last_session_id`)
- WebUI usability
  - dark theme
  - table-based identity views
  - policy match highlighting
  - session comparison and browsing
- README examples and PoC references updated to the VS Code forbidden-tool comparison case (`attested_poc/`)

### Changed

- Main PoC reference moved from the older Codex example to `attested_poc/` (VS Code forbidden-tool PASS/FAIL comparison + WebUI screenshots)
- Older Codex PoC documents and artifacts archived under `archive/` / `archive/v0.1.0/`

### Notes / Known Limitations

- `attestation.json` is currently kept as `latest` output in `.attest_run/attestations/latest/`
  - WebUI uses `ATTESTED_SUMMARY` to display per-session verification results when browsing older sessions
- `ATTESTED_WORKSPACE_OBSERVED` is cumulative (workspace-scoped)
  - session-scoped `ATTESTED_OBSERVED` is not generated in the current implementation
